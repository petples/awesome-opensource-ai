# Contributing to Awesome Open Source AI

Thank you for considering contributing to this curated list! We maintain a high quality bar to ensure every entry provides real value to the community.

## Quality Criteria

To be included, a project must meet **at least 2 of the following 3 criteria**:

| Criterion | Threshold | Why It Matters |
|-----------|-----------|----------------|
| **⭐ GitHub Stars** | 500+ | Indicates community adoption and trust |
| **📅 Project Age** | 6+ months old | Filters out abandoned experiments and hype-driven launches |
| **🔄 Active Development** | Commits within the last month | Ensures the project is maintained and evolving |

### Examples

| Project | Stars | Age | Active | Verdict |
|---------|-------|-----|--------|---------|
| New tool with 600⭐, 2 months old, active last week | ✅ | ❌ | ✅ | **Accept** — meets 2/3 |
| 300⭐ tool, 8 months old, dormant 3 months | ❌ | ✅ | ❌ | **Reject** — only 1/3 |
| 200⭐ tool, 1 year old, active yesterday | ❌ | ✅ | ✅ | **Accept** — meets 2/3 |

## What We're Looking For

### ✅ Strong Candidates

- **Proven tools** with real-world usage
- **Well-documented** projects with clear READMEs
- **Open source** (not just "free tier" proprietary tools)
- **Actively maintained** with responsive maintainers
- **Unique value** — not yet another thin wrapper

### 🚩 Red Flags

- Hype-heavy descriptions without substance
- "The agentic explosion is here" marketing fluff
- Single-contributor repos with 300⭐ claiming to be "enterprise-ready"
- Forks with minor changes masquerading as new projects
- Projects that are clearly AI-generated slop

## Submission Process

1. **Check the criteria first** — don't submit if it fails 2+ criteria
2. **Verify your entry** fits in the right category
3. **Include star badges** in your PR: `![GitHub stars](https://img.shields.io/github/stars/owner/repo?style=social)`
4. **Write a concise description** — one sentence, factual, no marketing speak

### Bad Description
> "The revolutionary AI-powered next-generation platform disrupting the industry with bleeding-edge agentic workflows"

### Good Description
> "Local LLM code completion for Vim using llama.cpp. Fast, privacy-first, no API key needed."

## Maintainer Override

The maintainer reserves the right to:
- **Reject** any submission that doesn't meet quality standards, even if technically meeting 2/3 criteria
- **Accept** exceptional projects that may fall short on stars but demonstrate unique innovation
- **Remove** existing entries if they become abandoned or quality degrades

## Questions?

Open an issue for discussion before submitting borderline cases. We're happy to help!
